Getting Started

  1. Create Systems (each assigned a row or use canvas to reposition).
  2. Drag systems on Canvas (tile) or in the list (handle) to reorder.
  3. Create Subgroups if needed.
  4. Add Nodes & assign System/Subgroup.
  5. Define incoming edges with the advanced connection controls.
  6. Click Update Diagram.

Systems & Rows

Row numbers group systems into horizontal bands (lane). Lower rows appear above. When all systems are row 0, the global Vertical/Horizontal toggle determines top-to-bottom or left-to-right orientation.

Canvas Drag & Drop

The Canvas shows system tiles. Drag a tile:

  • Horizontally ‚Üí reorder within the lane.
  • Vertically ‚Üí move to a different lane (row changes automatically).
Use "Compact Lanes" to remove empty vertical gaps and renumber lanes, or "Reset Layout" to linearize row 0.

Subgroups

Nested clusters inside a system.

Nodes & Connections

Connections represent inbound edges for the node.

Connection Styles

Types add arrow/line variants (solid, plain, dashed, dotted, thick, double, no-arrow versions). Customize arrow toggle, stroke width (1‚Äì6), and color. Patterns applied via linkStyle when safe.

Layout Rules

Per-system and subgroup layouts (vertical/horizontal) override global arrangement for internal node direction. Global arrangement only affects overall system stacking when single row is used.

Legend

Styled text overlay. Optionally include node descriptions in exports.

Export & Import

  • JSON: full state (includes systemPlacement).
  • PNG/PDF/SVG: diagram + legend (+ descriptions if option selected).
  • Import: missing systems/subgroups created automatically.

Troubleshooting

  • Edge style missing? Possibly safe fallback suppressed linkStyle; see console.
  • Turn on safe mode: EDGE_STYLE_SAFE_MODE = true; generateMermaid();
  • Arrows reversed? Remember edges are defined inbound (source ‚Üí current node).
